# Data Directory

This directory contains the raw and processed data files for the LinkedIn Job Analytics project.

## Directory Structure

```
data/
├── raw/              # Original unprocessed data
│   └── .gitkeep
├── processed/        # Cleaned and processed data
│   └── .gitkeep
└── README.md         # This file
```

## Data Files

### Raw Data (Place Here)
- **Final Dataset.xlsx**: Original LinkedIn job postings data
  - Should be placed in `data/raw/`
  - Not tracked in Git due to size
  - Contains job descriptions, titles, companies, locations, dates
- **translated_jobs.csv**: Intermediate file with translated text
  - Generated during data cleaning process
  - Contains title_translated and description_translated columns
  - Used as checkpoint before final processing

### Processed Data (Generated)
- **linkedin_data_final.xlsx**: Cleaned and translated data
  - Generated by `01_data_cleaning.ipynb`
  - Output location: `data/processed/`
  - Contains cleaned descriptions, extracted features

## Getting the Data

### Use Your Own Data
1. Place your LinkedIn job postings Excel file in `data/raw/`
2. Rename it to `Final Dataset.xlsx` or update the notebook path
3. Run `01_data_cleaning.ipynb`

## Data Schema

### Raw Data Columns
- `job_id`: Unique identifier for each job posting
- `title`: Job title
- `company`: Company name
- `place`: Location (City, State/Province, Country)
- `description`: Full job description (may contain HTML)
- `date`: Date posted
- `link`: URL to job posting

### Processed Data Columns
After running `01_data_cleaning.ipynb`:
- All raw columns +
- `title_clean`: Cleaned job title
- `description_clean`: HTML-free, translated description
- `country`: Extracted country
- `tools`: Extracted technical tools
- `education`: Education requirements
- `experience_years`: Years of experience required
- `experience_level`: Seniority level
- `job_type`: Full-time, Part-time, Contract, etc.
- `remote_onsite`: Remote, Hybrid, or Onsite

## Data Size Guidelines

- Raw data: Typically 10-50 MB
- Processed data: Typically 15-60 MB
- Keep files under 100 MB for GitHub (use Git LFS for larger files)

## Privacy & Ethics

- Ensure all data is collected ethically and legally
